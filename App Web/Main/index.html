GENERAL

El nombre del proyecto es Agenda Virtual y como su nombre indica se trata del conjunto de una aplicación móvil y una aplicación web que te ayudarán a gestionar y organizar tus asignaturas y tareas.
Al usar una base de datos en la nube las tareas y asignaturas de cada usuario están sincronizadas en las dos aplicaciones.

La base de datos nos la proporciona Firebase, plataforma en la nube que nos proporciona una serie de herramientas
las cuales facilitan el desarrollo de aplicaciones. En concreto la herramienta que nos permite alojar y gestionar
la base de datos en la nube se llama Firestore. Se trata de una base de datos documental , un tipo de las noSQL,
a diferencia de las bases de datos relacionales una entidad o tabla equivale a una colección,
cada registro o fila de la tabla equivale a un documento y los atributos o columnas de la tabla son sustituidos por propiedades.
La estructura es formato JSON lo que permite manejar grandes cantidades de datos con mayor ligereza que las SQL.

PÁGINA WEB

APLICACIÓN MÓVIL

Aplicación Android para gestionar y organizar tus asignaturas y tareas desde tu teléfono móvil.

-Registrarse y loguearse en la aplicación.
-Desloguearse de la aplicación.
-Crear, Visualizar, Actualizar y Eliminar tareas de la base de datos.
-Crear, Visualizar y Eliminar asignaturas de la base de datos.
-Cambiar de tema (claro/oscuro).

Lenguaje de programación: Kotlin.
    Dependencias de kotlin utilizadas:
        Corrutinas (procesos asíncronos).
        ViewModel (intermediario entre UI y lógica de la aplicación).
        LiveData (clase de contenedor de datos observables).
        Fragment (Pantallas de la UI).
        Navigation (Navegación entre fragmentos).
    Firebase:
        Analytics (Análisis de las acciones de los usuarios en la app).
        Auth (Registro y login del usuario con correo y contraseña).
        Auth-Google (Registro y login del usuario con Google).
        FireStore (Acceso a la base de datos).

Android xml.
    Vistas utilizadas:
        -Contenedor de fragmentos.
            Utilizado para contener las diferentes pantallas de la app.
        -Botones.
        -Vistas de texto plano.
        -Vistas de texto editable (inputs).
        -RecyclerView (deslizar hacia abajo hasta que no queden elementos que mostrar).
            Utilizado para mostrar la lista de asignaturas y tareas.
        -Cards o tarjetas (contiene otras vistas, da la sensación de que flota).
            Utilizado para mostrar cada elemento de la lista de asignaturas y tareas.
        -Datepicker (calendario para seleccionar una fecha).
            Utilizado para seleccionar la fecha de entrega de la tarea.
        -FloatingActionButton (botón flotante).
            Utilizado para insertar y modificar asignaturas y tareas.
        -Switch (interruptor).
            Utilizado para marcar si la tarea está completa o no.

Requisitos: Tener descargado e instalado Java
En mi caso tuve que extraer la huella digital (SHA-1) de la apk de producción generada e introducirla en la configuración general del proyecto en Firebase.
cd C:\Program Files\Java\<jdk_versión>\bin
-list -v -keystore <nombre_keystore> -alias <nombre_alias>
Si no hago esto no funcionaría el login con Google, esto se debe a que Firebase es mi proveedor de autenticación con Google.
Explicación detallada: https://developer.android.com/studio/publish/app-signing?hl=es-419#generate-key
