GENERAL

The name of the project is Agenda Virtual and as its name suggests, it is a set of of a mobile application and a web application that will help you manage and organize your subjects and homework. By using a cloud database, each user's tasks and assignments are synchronized between the two applications.

The database is provided by Firebase, cloud platform that provides us with a variety of tools and services which facilitate the development of applications. In particular, the tool that allows us to host and manage the database is called Firestore. This is a documentary database, a type of the noSQL, Unlike relational databases, an entity or table is equivalent to a collection, each record or row of the table is equivalent to a document and the attributes or columns of the table are replaced by properties. The structure is in JSON format, which makes it possible to handle large amounts of data more lightly than SQL.

The organization has been done with Trello, a multiplatform application where you can organize any type of activity in a very visual way with a Canvas board which allows you to drag labels, change their color, add a description and much more.

MOBILE APPLICATION

Android application to manage and organize your subjects and homework from your cell phone.

-Register and login in the application.
-Logout of the application.
-Create, Read, Update and Delete database tasks.
-Create, Read and Delete subjects from the database.
-Change theme (light/dark).

Programming language: Kotlin.

    Kotlin dependencies used:

        -Corrutines (asynchronous processes).
        -ViewModel (intermediary between UI and application logic).
        -LiveData (observable data container class).
        -Fragment (UI screens).
        -Navigation (Navigation between fragments).

    Firebase:

        -Analytics (Analysis of user actions in the app).
        -Auth (User registration and login with email and password).
        -Auth-Google (User registration and login with Google).
        -FireStore (Database access).

Android xml.

    Views used:

        -Fragment container.
                Used to contain the different screens of the app.
        -Buttons.
        -Plain text views.
        -Editable text views (inputs).
        -RecyclerView (scroll down until there are no items left to display).
                Used to display the list of subjects and tasks.
        -Cards (contains other views, gives the sensation of floating).
                Used to display each item in the list of subjects and tasks.
        -Datepicker (calendar to select a date).
                Used to select the submit date of the task.
        -FloatingActionButton (floating button).
                Used to add and modify subjects and tasks.
        -Switch (interruptor).
                Used to mark if the task is completed or not.

In my case I had to extract the fingerprint (SHA-1) of the generated production apk and insert it in the general configuration of the firebase project.
cd C:\Program Files\Java\<jdk_versión>\bin
-list -v -keystore <keystore_name> -alias <alias_name>
If I don't do this the login with Google will not work, this is because Firebase is my Google authentication provider.
