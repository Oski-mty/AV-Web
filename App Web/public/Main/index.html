GENERAL

The name of the project is Agenda Virtual and as its name suggests, it is a set of of a mobile application and a web application that will help you manage and organize your subjects and homework. By using a cloud database, each user's tasks and assignments are synchronized between the two applications.

The database is provided by Firebase, cloud platform that provides us with a variety of tools and services which facilitate the development of applications. In particular, the tool that allows us to host and manage the database is called Firestore. This is a documentary database, a type of the noSQL, Unlike relational databases, an entity or table is equivalent to a collection, each record or row of the table is equivalent to a document and the attributes or columns of the table are replaced by properties. The structure is in JSON format, which makes it possible to handle large amounts of data more lightly than SQL.

The organization has been done with Trello, a multiplatform application where you can organize any type of activity in a very visual way with a Canvas board which allows you to drag labels, change their color, add a description and much more.

During development, I encountered two main challenges:

First, I faced difficulties with CSS selectors, as they affected unwanted elements, generating chaos in the styling of my page. To solve this problem, I decided to replace the selectors with classes, which allowed me to have more control over the styles and avoid unwanted conflicts.

The second challenge was related to the management of subject identifiers (IDs). Initially, I used the name of the subjects as the ID, but this had a drawback. The IDs are immutable, which means that they cannot be modified. Therefore, if a user changed the name of a subject, the associated ID would remain the previous name. This created problems when trying to add another subject with the modified name, since there was a conflict with the previous ID. To overcome this difficulty, I implemented the generation of auto-generated codes as subject IDs. This ensured that each subject had a unique ID and avoided duplication problems when changing names.

These challenges gave me the opportunity to improve my approach and learn new strategies for solving technical problems. As I continue to develop, I am committed to finding efficient solutions and maintaining the integrity of my code and data.

UiPath Robot

What is UiPath software?
Uipath is a software for RPA (Robotic Process Automation) that allows companies to configure robots for specific processes. The UI also refers to the interface with which the user interacts with the robot.
The robot I have made collects data from Classroom's unread emails, then navigates to the web application and adds the tasks automatically.
